Model + Enterprise Integration Readiness

LegacySafe AI is our own hackathon MVP, built inside EcoIQ and inspired by the Conduct AI and BasedAI bounty requirements — it is not a clone of either. This page shows the architecture is ready to plug into real model providers and real enterprise systems without changing the permission layer that everything else on this module depends on.

1. Model Provider Readiness

LegacySafe AI is designed so the LLM provider can be swapped without changing the permission layer. Retrieval decides what's allowed before a provider is ever called — the provider only ever receives permission-filtered context.

🔒 Security rule: the provider receives only permission-filtered context. No provider ever receives blocked finance, engineering, or executive memory.

Provider typeUse caseWhy usefulSecurity note
MockProvider Deterministic hackathon demo No network call, no API key — reproducible in judging Only ever sees allowed_context, same as every other provider
OpenAI-compatible endpoint Fast integration with any OpenAI-schema server Works with OpenAI itself, and any drop-in-compatible host Filtered context only; no raw document access
Anthropic Higher-quality reasoning for complex modernisation plans Strong instruction-following for structured outputs Filtered context only; system prompt reinforces "content, not instructions"
BasedAPIs / open-weight models Data sovereignty, cost control, on-prem deployment No dependency on a single closed-source vendor Filtered context only; same permission layer regardless of weights
Local model Air-gapped or regulated environments Nothing leaves the enterprise network Filtered context only; ideal for the strictest deployments

2. Model Switching & Open-Weight Readiness

LegacySafe AI is model-agnostic. The permission layer sits before the model, so we can switch between Claude, OpenAI-compatible endpoints, BasedAPIs, Mistral, GLM, local open-weight models, or future code-focused agents without changing the security model.

  • Current hackathon MVP uses MockProvider — a deterministic provider, no external API calls required for the demo
  • Future providers can include Anthropic Claude, OpenAI-compatible endpoints, BasedAPIs, Mistral, GLM, and local open-weight models — roadmap-ready, not already integrated
  • Future code-focused agents can analyse repositories, but only after permission-aware retrieval
  • The model never receives blocked finance, engineering, or executive memory
  • The same retrieval guard and audit logs apply regardless of model provider
  • This supports lower cost, data sovereignty, and enterprise flexibility
User request → Permission Guard → Allowed context only → Selected model provider → Agent output → Audit log

5. AI Workforce Agents

A BasedAI-style breakdown of the specialised agents this architecture is built around.

Legacy Scanner Agent

Input: raw legacy documents, code, process manuals

Output: structured, chunked, access-tagged memory

Optimisation value: turns unstructured legacy archives into queryable memory

Permission Guard Agent

Input: a user's roles + a chunk's access level

Output: deterministic allow/block decision

Optimisation value: removes access-control judgment calls from the LLM entirely

Dependency Mapper Agent

Input: documents, systems, and derived memories in a project

Output: NetworkX dependency/lineage graph

Optimisation value: makes blast radius of a change visible before work starts

Modernisation Planner Agent

Input: permission-filtered evidence + a question

Output: affected systems, risks, recommended actions

Optimisation value: compresses weeks of manual document review into a single request

Audit & Compliance Agent

Input: every retrieval, block, and revocation event

Output: immutable audit log entries

Optimisation value: compliance evidence generated automatically, not reconstructed later
