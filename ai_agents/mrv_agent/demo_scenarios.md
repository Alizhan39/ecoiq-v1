# MRV Agent — Demo Scenarios

## Investor demo scenario

**Audience:** sovereign fund deciding whether a clean heating pilot's impact
claims are safe to cite in a board pack.

MRV Agent shows the project sitting at "MRV In Review" — baseline and
after-data both exist, but a reviewer has not yet signed off — with a clear
statement that the board pack must use "estimated," not "verified," language
until that review completes.

**Why it lands:** it demonstrates EcoIQ will not let a good story get ahead
of the evidence, which is exactly what a serious capital allocator wants to see.

## Enterprise / government use case

**Audience:** a government body deciding which pilot projects are ready for a
public "clean heating" announcement.

MRV Agent's public_reporting_ready flag gives the government a clear,
defensible gate — no project gets announced publicly until MRV Verified and
consent/privacy checks both pass.

## Islamic ethical finance / Amanah alignment

Maqasid/Mizan public impact wording is always routed through human approval —
MRV Agent flags the requirement but never approves such wording itself,
keeping the ethical determination with a qualified human reviewer.

## MRV / audit trail requirements

This is the audit-trail agent by design: every stage transition is backed by
specific baseline/after-data evidence, methodology, and (for MRV Verified) a
recorded human approval — the full trail an external auditor would need.
