# Photo / Visual Evidence Agent — Demo Scenarios

## Investor demo scenario

**Audience:** impact investor reviewing a boiler modernisation pipeline.

1. Field engineer uploads boiler house photos via Mobile / iPad Inspection Mode.
2. Photo / Visual Evidence Agent flags possible insulation gaps and a missing
   smart meter — each labelled "needs verification."
3. Investor sees the visual evidence trail feeding directly into the Asset
   Passport, with an explicit statement that engineering sign-off is still
   required before any recommendation proceeds.

**Why it lands:** it shows visual AI is used to speed up triage, not to
replace the engineer — a distinction investors and insurers care about.

## Enterprise / government use case

**Audience:** a mining regulator reviewing tailings dam safety across a
portfolio of sites.

Photo / Visual Evidence Agent processes aerial/drone imagery across sites,
flagging possible erosion or seepage for expert follow-up — giving the
regulator a triage list rather than a final safety verdict, which stays with
qualified inspectors.

## Islamic ethical finance / Amanah alignment

Visible harm indicators (smoke, spills, unsafe conditions) are explicitly
routed to Governance Agent for human review, reflecting the Maqasid
principle of preventing harm (la darar) — this agent never resolves a safety
concern on its own.

## MRV / audit trail requirements

Every finding is timestamped and linked to its source photo/frame, so an MRV
or audit reviewer can trace a "before" condition claim back to the exact
image it came from.
